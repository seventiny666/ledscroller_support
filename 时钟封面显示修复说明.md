# 时钟封面显示修复说明

## 问题描述

用户上传了时钟封面图片（`clock_1.jpeg`），但在首页"数字时钟"分类中不显示封面图片。

## 问题原因

在`TemplateSquareViewController.swift`的`LEDItem`扩展中，`imageName`属性只处理了三种模版类型：
- `neon` - 霓虹灯看板
- `idol` - 偶像应援
- `led` - LED横幅

但没有处理`clock`（时钟）类型，导致翻页时钟卡片无法加载封面图片。

## 修复方案

在`imageName`属性中添加对翻页时钟的支持：

```swift
var imageName: String? {
    // 图片命名规则：category_number.png
    // 例如：neon_1.png, idol_2.png, led_3.png, clock_1.png
    if isNeonTemplate {
        return "neon_\(id.replacingOccurrences(of: "neon-", with: ""))"
    } else if isIdolTemplate {
        return "idol_\(id.replacingOccurrences(of: "idol-", with: ""))"
    } else if isLEDTemplate {
        return "led_\(id.replacingOccurrences(of: "led-", with: ""))"
    } else if isFlipClock {
        // 翻页时钟使用 clock_1
        return "clock_1"  // ✅ 新增
    }
    return nil
}
```

## 修复效果

### 1. 图片加载逻辑
```swift
func configure(with item: LEDItem) {
    titleLabel.text = item.text
    
    // 尝试加载图片
    if let imageName = item.imageName, !imageName.isEmpty {
        imageView.image = UIImage(named: imageName)  // ✅ 现在可以加载 clock_1
    } else {
        // 使用占位颜色
        imageView.image = nil
        imageView.backgroundColor = UIColor(hex: item.backgroundColor)
    }
}
```

## 技术细节

### 图片加载流程
```
1. getItems(for: .clock)
   ↓
2. 返回 LEDItem (id: "flip-clock-special", isFlipClock: true)
   ↓
3. configure(with: item)
   ↓
4. item.imageName
   ↓
5. isFlipClock == true → return "clock_1"
   ↓
6. UIImage(named: "clock_1")
   ↓
7. 从 Assets.xcassets 加载图片
   ↓
8. 显示在 imageView 中
```

### 为什么其他特殊效果不需要封面

其他特殊效果卡片（烟花、爱心流星雨）在"其他分类"中显示，它们：
- 不是模版类型（isNeonTemplate/isIdolTemplate/isLEDTemplate 都是 false）
- 不需要封面图片
- 直接使用背景色显示

只有模版类型和翻页时钟需要封面图片。
