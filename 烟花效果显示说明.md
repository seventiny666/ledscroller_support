# 烟花效果显示说明

## 烟花效果位置

烟花效果应该在首页的**"其他分类"**中显示。

## 两种烟花效果

### 1. 烟花效果（FireworksViewController）
- **卡片文字**: 🎆 烟花
- **ID**: `fireworks-special`
- **效果**: 烟花从底部向上发射，然后在顶部炸开
- **方向**: 竖屏

### 2. 烟花绽放效果（FireworksBloomViewController）
- **卡片文字**: 🎇 烟花绽放
- **ID**: `fireworks-bloom-special`
- **效果**: 烟花在屏幕中心绽放，多彩粒子向四周扩散
- **方向**: 竖屏

## 为什么可能看不到

### 原因1：预设数据未加载
烟花卡片定义在 `LEDItem.swift` 的 `defaultItems` 中，需要确保这些预设数据被加载。

**检查方法**：
1. 运行应用
2. 进入"模版"页面
3. 滚动到最底部的"其他分类"
4. 查看是否有烟花卡片

### 原因2：数据被过滤
"其他分类"的过滤逻辑：
```swift
case .other:
    return allItems.filter { 
        !$0.isFlipClock && 
        !$0.isNeonTemplate && 
        !$0.isIdolTemplate && 
        !$0.isLEDTemplate 
    }
```

烟花卡片**不会**被过滤掉，因为：
- `isFireworks` 和 `isFireworksBloom` 不在过滤条件中
- 它们应该正常显示

### 原因3：卡片被遮挡
如果"其他分类"的卡片很多，烟花卡片可能在底部被遮挡。

**已修复**：
- Row高度已增加到350px
- 底部contentInset增加到20px
- 确保所有卡片完全显示

## 可能的问题和解决方案

### 问题1：预设数据未初始化
**症状**：首页"其他分类"为空或只有用户创建的卡片

**解决方案**：
确保 `LEDDataManager` 正确加载了预设数据。检查 `loadItems()` 方法是否返回了 `defaultItems`。

### 问题2：卡片被过滤
**症状**：其他卡片显示，但烟花卡片不显示

**解决方案**：
检查过滤逻辑，确保 `isFireworks` 和 `isFireworksBloom` 不在过滤条件中。

### 问题3：卡片被遮挡
**症状**：滚动到底部看不到最后的卡片

**解决方案**：
- ✅ 已增加Row高度到350px
- ✅ 已增加底部contentInset到20px
- ✅ 确保所有卡片完全显示

### 问题4：数据未保存
**症状**：重启应用后烟花卡片消失

**解决方案**：
预设卡片应该每次都从 `defaultItems` 加载，不依赖持久化存储。
