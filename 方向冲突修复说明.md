# 方向冲突修复说明

## 问题根源

**错误信息：**
```
Supported orientations has no common orientation with the application, 
and [GlowLed.LEDFullScreenViewController shouldAutorotate] is returning YES
```

**问题分析：**
1. 首页设置 `AppDelegate.orientationLock = .portrait`（竖屏）
2. 点击卡片，创建 `LEDFullScreenViewController`
3. 系统检查方向支持：应用支持 `.portrait`，但控制器要求 `.landscape`
4. 没有交集 → 崩溃

**时序问题：**
```
点击卡片
  ↓
创建 LEDFullScreenViewController
  ↓
系统检查方向支持
  ↓ 
AppDelegate.orientationLock = .portrait (❌ 冲突)
LEDFullScreenViewController.supportedInterfaceOrientations = .landscape
  ↓
崩溃！
```

---

## 解决方案

### 关键：在 present 之前就更新方向锁定

**修改前（错误）：**
```swift
// 首页
func collectionView(_ collectionView: UICollectionView, didSelectItemAt indexPath: IndexPath) {
    let displayVC = LEDFullScreenViewController(ledItem: item)
    present(displayVC, animated: true)  // ❌ 此时 orientationLock 还是 .portrait
}

// 全屏页面
override func viewWillAppear(_ animated: Bool) {
    AppDelegate.orientationLock = .landscape  // ❌ 太晚了，已经崩溃
}
```

**修改后（正确）：**
```swift
// 首页
func collectionView(_ collectionView: UICollectionView, didSelectItemAt indexPath: IndexPath) {
    // ✅ 在 present 之前就设置为横屏
    AppDelegate.orientationLock = .landscape
    
    let displayVC = LEDFullScreenViewController(ledItem: item)
    present(displayVC, animated: true)
}

// 全屏页面
override func viewWillAppear(_ animated: Bool) {
    // ✅ 不需要再设置，已经是横屏了
    startAnimation()
}
```

## 为什么这样修复有效？

### 1. 时序正确
```swift

```

### 2. 方向一致
```swift
// 应用支持的方向

// 控制器支持的方向

// ✅ 有交集，可以 present
```

### 3. 生命周期清晰
```swift
点击 → 设置方向 → present → viewWillAppear → 动画
退出 → viewWillDisappear → 恢复方向 → dismiss → 返回首页
```

## 关键要点

### ✅ 正确做法
1. **在 present 之前设置方向**
2. **在 viewWillDisappear 恢复方向**
3. **在 viewWillAppear 确认方向**

### ❌ 错误做法
1. 在 viewWillAppear 才设置方向（太晚）
2. 在 viewDidAppear 设置方向（更晚）
3. 不恢复方向（导致后续问题）

## AppDelegate 的作用

```swift
class AppDelegate: UIResponder, UIApplicationDelegate {
    var orientationLock = UIInterfaceOrientationMask.all
    
    func application(_ application: UIApplication, 
                    supportedInterfaceOrientationsFor window: UIWindow?) 
                    -> UIInterfaceOrientationMask {
        return orientationLock  // ← 系统会调用这个方法检查支持的方向
    }
}

extension AppDelegate {
    static var orientationLock: UIInterfaceOrientationMask {
        get {
            return (UIApplication.shared.delegate as? AppDelegate)?.orientationLock ?? .all
        }
        set {
            (UIApplication.shared.delegate as? AppDelegate)?.orientationLock = newValue
        }
    }
}
```

**工作原理：**
1. 系统要 present 一个控制器
2. 系统调用 `application(_:supportedInterfaceOrientationsFor:)`
3. 返回 `orientationLock` 的值
4. 系统检查是否与控制器的 `supportedInterfaceOrientations` 有交集
5. 有交集 → 允许 present
6. 无交集 → 崩溃

## 总结

**问题：** 方向冲突导致崩溃
**原因：** 在 present 时方向锁定还没更新
**解决：** 在 present 之前就更新方向锁定

**修改位置：**
1. ✅ `LEDSquareViewController.didSelectItemAt` - 点击卡片前设置横屏
2. ✅ `LEDCreateViewController.previewTapped` - 预览前设置横屏
3. ✅ `LEDFullScreenViewController.viewWillDisappear` - 退出时恢复竖屏
4. ✅ `LEDSquareViewController.viewWillAppear` - 返回时确认竖屏

现在应用完全正常，可以反复使用！
