# 内购系统更新日志

## 2026-03-20

### ✅ 添加交易去重功能

**背景**:
为了防止在极端情况下（App崩溃、网络异常、系统重启等）重复处理同一笔交易，添加了交易去重机制。

**实现内容**:

1. **新增属性**
   - `processedTransactionsKey`: 存储已处理交易ID的键
   - `processedTransactions`: 已处理交易ID的集合（Set）

2. **新增方法**
   - `isTransactionProcessed(_:)`: 检查交易是否已处理
   - `markTransactionAsProcessed(_:)`: 标记交易为已处理
   - `cleanupOldTransactions()`: 清理过期交易记录（可选）

3. **修改逻辑**
   - 在 `paymentQueue(_:updatedTransactions:)` 方法中：
     - 购买成功前检查交易是否已处理
     - 恢复购买前检查交易是否已处理
     - 处理成功后标记交易ID
     - 自动限制记录数量（最多100个）

**代码位置**:
- 文件: `LedScreen/TemplateSquareViewController.swift`
- 类: `VIPManager`
- 行数: ~400-430 (交易去重方法), ~2140-2180 (使用去重逻辑)

**测试建议**:
1. 购买后立即杀掉 App，重启验证不会重复处理
2. 连续多次恢复购买，验证只处理一次
3. 查看日志确认交易ID被正确记录

**日志输出**:
```
✅ 交易已标记为已处理: 1000000123456789
⚠️ 交易已处理过，跳过: 1000000123456789
```
