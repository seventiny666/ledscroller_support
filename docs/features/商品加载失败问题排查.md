# 商品加载失败问题排查

## 🔍 问题现象

VIP订阅界面显示弹窗：
```
Tip
Loading subscription options, please try again later
```

订阅选项显示为空或价格为 $0.00

## 📋 问题原因

商品列表加载失败，`VIPManager.shared.products` 数组为空。

可能的原因：

### 1. ❌ 产品未在 App Store Connect 配置（最常见）

**检查方法**:
- 登录 [App Store Connect](https://appstoreconnect.apple.com)
- 进入你的 App
- 点击"功能" → "App 内购买项目"
- 检查是否存在以下产品：
  - `com.ledscreen.vip.weekly` (周订阅)
  - `com.ledscreen.vip.monthly` (月订阅)
  - `com.ledscreen.vip.yearly` (年订阅)

**解决方案**:
参考 `App内购配置指南.md` 创建订阅产品

### 2. ❌ 产品状态不正确

**检查方法**:
- 在 App Store Connect 中检查产品状态
- 必须是"准备提交"或"等待审核"状态
- 不能是"草稿"状态

**解决方案**:
- 完善产品信息
- 提交产品审核

### 3. ❌ Bundle ID 不匹配

**检查方法**:
- Xcode 中的 Bundle ID: `com.ledscreen.app`
- App Store Connect 中的 Bundle ID 必须一致
- 产品 ID 前缀必须匹配: `com.ledscreen.vip.*`

**解决方案**:
确保 Bundle ID 完全一致

### 4. ❌ 协议未签署

**检查方法**:
- App Store Connect → "协议、税务和银行业务"
- 检查"付费 App 协议"是否已签署

**解决方案**:
签署所有必需的协议

### 5. ❌ 网络问题

**检查方法**:
- 查看 Xcode 控制台日志
- 是否有网络错误信息

**解决方案**:
- 检查网络连接
- 尝试切换网络（WiFi/4G）

### 6. ❌ 沙盒账号问题

**检查方法**:
- 设置 → App Store → 沙盒账号
- 检查是否登录了沙盒测试账号

**解决方案**:
- 退出当前沙盒账号
- 重新登录
- 或创建新的沙盒测试账号

## 🔧 调试步骤

### 步骤 1: 查看日志

在 Xcode 控制台查找以下日志：

```swift
// 商品加载成功
成功加载 3 个产品

// 商品加载失败
成功加载 0 个产品
产品请求失败: [错误信息]
```

### 步骤 2: 添加调试代码

在 `VIPManager` 的 `productsRequest` 方法中添加：

```swift
func productsRequest(_ request: SKProductsRequest, didReceive response: SKProductsResponse) {
    DispatchQueue.main.async {
        print("🔍 收到产品响应")
        print("🔍 有效产品数量: \(response.products.count)")
        print("🔍 无效产品ID: \(response.invalidProductIdentifiers)")
        
        for product in response.products {
            print("🔍 产品: \(product.productIdentifier)")
            print("   - 名称: \(product.localizedTitle)")
            print("   - 价格: \(product.price)")
            print("   - 货币: \(product.priceLocale.currencyCode ?? "unknown")")
        }
        
        // 原有代码...
    }
}
```

### 步骤 3: 检查无效产品ID

如果日志显示无效产品ID，说明：
- 产品未在 App Store Connect 创建
- 产品ID拼写错误
- Bundle ID 不匹配

### 步骤 4: 验证产品ID

确认代码中的产品ID：
```swift
enum ProductID: String, CaseIterable {
    case weekly = "com.ledscreen.vip.weekly"
    case monthly = "com.ledscreen.vip.monthly"
    case yearly = "com.ledscreen.vip.yearly"
}
```

必须与 App Store Connect 中的完全一致！

## ✅ 解决方案（按优先级）

### 🔴 优先级 1: 配置 App Store Connect

1. 登录 App Store Connect
2. 创建订阅群组
3. 创建 3 个订阅产品
4. 设置价格和周期
5. 提交审核

**详细步骤**: 参考 `App内购配置指南.md`

### 🟡 优先级 2: 检查沙盒账号

1. 设置 → App Store → 退出登录
2. 运行 App
3. 首次购买时会提示登录沙盒账号
4. 使用在 App Store Connect 创建的沙盒测试账号

### 🟢 优先级 3: 清理缓存

1. 删除 App
2. 重启设备
3. 重新安装 App
4. 重新测试

## 📱 测试流程

### 正确的测试流程：

1. **配置 App Store Connect**
   - 创建订阅产品
   - 等待产品状态变为"准备提交"

2. **创建沙盒测试账号**
   - App Store Connect → "用户和访问" → "沙盒测试员"
   - 创建测试账号

3. **设备准备**
   - 退出 App Store 账号
   - 不要在设置中登录沙盒账号

4. **运行测试**
   - 在 Xcode 中运行 App
   - 打开 VIP 订阅界面
   - 查看日志确认商品加载成功
   - 点击购买时会提示登录
   - 使用沙盒测试账号登录

## 🎯 快速检查清单

- [ ] App Store Connect 中已创建订阅产品
- [ ] 产品状态为"准备提交"或更高
- [ ] Bundle ID 完全匹配
- [ ] 产品 ID 拼写正确
- [ ] 已签署付费 App 协议
- [ ] 已创建沙盒测试账号
- [ ] 设备已退出 App Store 账号
- [ ] 网络连接正常
- [ ] Xcode 控制台显示"成功加载 3 个产品"

## 📊 常见错误信息

### "Cannot connect to iTunes Store"
- **原因**: 网络问题或沙盒服务器问题
- **解决**: 检查网络，稍后重试

### "Invalid Product Identifier"
- **原因**: 产品ID不存在或拼写错误
- **解决**: 检查 App Store Connect 配置

### "成功加载 0 个产品"
- **原因**: 产品未配置或状态不正确
- **解决**: 完成 App Store Connect 配置

## 💡 提示

1. **首次配置需要时间**
   - 产品创建后可能需要几小时才能在沙盒环境生效
   - 耐心等待，不要频繁重试

2. **沙盒环境限制**
   - 沙盒环境可能不稳定
   - 如果一直失败，尝试第二天再测试

3. **生产环境测试**
   - 通过 TestFlight 测试
   - 或提交审核后在正式环境测试

## 📞 需要帮助？

如果按照以上步骤仍然无法解决：

1. 检查 Xcode 控制台的完整日志
2. 截图 App Store Connect 的产品配置
3. 确认 Bundle ID 和产品 ID
4. 查看是否有错误提示
