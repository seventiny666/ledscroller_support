# 预设卡片显示修复完成

## 问题分析

用户报告在首页"其他分类"中只能看到4个卡片：
- 💖 爱心流星雨
- HAPPY BIRTHDAY
- HAPPY NEW YEAR
- MERRY CHRISTMAS

缺失的5个卡片：
- 🎆 烟花
- 🎇 烟花绽放
- I LOVE U
- MARRY ME
- I ❤️ U

## 根本原因

在`LEDItem.swift`的`getDefaultItems()`方法中，部分预设卡片没有指定明确的ID：

```swift
// ❌ 问题代码：没有指定ID
LEDItem(
    text: "HAPPY BIRTHDAY",  // 会生成随机UUID
    fontSize: 50,
    ...
)
```

当没有指定ID时，`init`方法会生成一个随机UUID：
```swift
init(id: String = UUID().uuidString, ...) {
    self.id = id
    ...
}
```

这导致每次调用`getDefaultItems()`时，这些卡片都会获得不同的UUID。在`loadItems()`的合并逻辑中：

```swift
for defaultItem in defaultItems {
    if !mergedItems.contains(where: { $0.id == defaultItem.id }) {
        mergedItems.append(defaultItem)
    }
}
```

由于ID每次都不同，合并逻辑无法识别这些卡片是否已存在，导致：
1. 如果UserDefaults中已有旧数据，新的预设卡片无法被添加
2. 用户只能看到最初保存的4个卡片

## 修复方案

为所有预设卡片添加明确的固定ID：

```swift
private func getDefaultItems() -> [LEDItem] {
    return [
        LEDItem(
            id: "love-rain-special",
            text: "💖 爱心流星雨",
            ...
```

### 2. 合并逻辑正常工作
```swift
func loadItems() -> [LEDItem] {
    guard let data = userDefaults.data(forKey: key),
          let savedItems = try? JSONDecoder().decode([LEDItem].self, from: data) else {
        return getDefaultItems()  // 新用户：返回所有10个预设卡片
    }
    
    let defaultItems = getDefaultItems()
    var mergedItems = savedItems
    
    // 添加缺失的默认卡片
    for defaultItem in defaultItems {
        if !mergedItems.contains(where: { $0.id == defaultItem.id }) {
            mergedItems.append(defaultItem)  // ✅ 现在可以正确识别和添加
        }
    }
    
    return mergedItems  // 旧用户：保留旧数据 + 添加新的预设卡片
}
```

## 技术细节

### ID命名规范
- 特殊效果：`{effect-name}-special`（如 `fireworks-special`）
- 普通预设：`{text-slug}-default`（如 `happy-birthday-default`）
- 用户创建：随机UUID（如 `A1B2C3D4-...`）

### 为什么需要固定ID
1. **唯一性识别**：通过ID判断卡片是否已存在
2. **数据合并**：新旧数据合并时不会重复
3. **版本兼容**：应用更新后能正确添加新的预设卡片
4. **用户数据保护**：不会覆盖用户创建的卡片

### 合并策略
```
旧用户数据（UserDefaults）:
- HAPPY BIRTHDAY (随机UUID-1)
- HAPPY NEW YEAR (随机UUID-2)
- MERRY CHRISTMAS (随机UUID-3)
- 💖 爱心流星雨 (love-rain-special)

新的预设数据（getDefaultItems）:
- 💖 爱心流星雨 (love-rain-special)
- HAPPY BIRTHDAY (happy-birthday-default)
- HAPPY NEW YEAR (happy-new-year-default)
- MERRY CHRISTMAS (merry-christmas-default)
- MARRY ME (marry-me-default)
- I ❤️ U (i-heart-u-default)
- I LOVE U (i-love-u-default)
- 🎆 烟花 (fireworks-special)
- 🎇 烟花绽放 (fireworks-bloom-special)
- 🕐 翻页时钟 (flip-clock-special)

合并结果：
- HAPPY BIRTHDAY (随机UUID-1) ← 保留旧数据
- HAPPY NEW YEAR (随机UUID-2) ← 保留旧数据
- MERRY CHRISTMAS (随机UUID-3) ← 保留旧数据
- 💖 爱心流星雨 (love-rain-special) ← 已存在，不重复添加
- MARRY ME (marry-me-default) ← 新增
- I ❤️ U (i-heart-u-default) ← 新增
- I LOVE U (i-love-u-default) ← 新增
- 🎆 烟花 (fireworks-special) ← 新增
- 🎇 烟花绽放 (fireworks-bloom-special) ← 新增
- 🕐 翻页时钟 (flip-clock-special) ← 新增
```

注意：旧的HAPPY BIRTHDAY等卡片会保留（因为ID不同），但新的预设卡片也会被添加。这样用户不会丢失任何数据。
