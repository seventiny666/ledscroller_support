# 图片添加详细指南

## 快速回答

### 图片存放位置
```
GlowLed/Assets.xcassets/
```

### 命名规则
- 霓虹灯看板：`neon_1`, `neon_2`, `neon_3`, `neon_4`
- 偶像应援：`idol_1`, `idol_2`, `idol_3`, `idol_4`
- LED屏幕：`led_1`, `led_2`, `led_3`, `led_4`

**注意**：在 Assets.xcassets 中，图片名称不需要 `.png` 后缀！

## 详细步骤

### 方法1：通过Xcode添加（推荐）

#### 步骤1：打开Assets.xcassets
1. 在Xcode中打开 `GlowLed.xcodeproj`
2. 在左侧项目导航器中找到 `GlowLed` 文件夹
3. 点击 `Assets.xcassets` 文件夹

#### 步骤2：添加图片
1. 将准备好的图片文件直接拖拽到 Assets.xcassets 窗口中
2. 或者右键点击空白处 → "New Image Set"

#### 步骤3：重命名
1. 选中新添加的 Image Set
2. 在右侧属性面板中，将名称改为对应的名称
   - 例如：`neon_1`（不要加 .png）

#### 步骤4：设置图片
1. 将图片拖拽到 `1x`、`2x` 或 `3x` 的位置
2. 通常只需要放在 `1x` 位置即可
3. 如果有高清图，可以放在 `2x` 和 `3x`

### 方法2：直接拖拽（更快）

1. 打开 Xcode 项目
2. 点击 `Assets.xcassets`
3. 将所有图片文件一次性拖拽进去
4. Xcode 会自动创建 Image Set
5. 逐个重命名为正确的名称

## 图片规格要求

### 尺寸
- **宽高比**：16:9（必须）
- **推荐尺寸**：
  - 1x: 640×360 px
  - 2x: 1280×720 px
  - 3x: 1920×1080 px
- **最小尺寸**：640×360 px

### 文件格式
- **推荐格式**：PNG（支持透明）
- **备选格式**：JPG（文件更小）
- **颜色模式**：RGB
- **位深度**：8位

### 文件大小
- **单张图片**：建议 < 500KB
- **总大小**：12张图片 < 6MB
- **优化建议**：使用 TinyPNG 或 ImageOptim 压缩

## 代码中的使用

### 图片加载方式
```swift
// 在 TemplateItemCell 中
if let imageName = item.imageName, !imageName.isEmpty {
    imageView.image = UIImage(named: imageName)
} else {
    // 显示占位符颜色
    imageView.image = nil
    imageView.backgroundColor = UIColor(hex: item.backgroundColor)
}
```

### 图片名称获取
```swift
// 在 LEDItem 扩展中
var imageName: String? {
    if isNeonTemplate {
        return "neon_\(id.replacingOccurrences(of: "neon-", with: ""))"
    } else if isIdolTemplate {
        return "idol_\(id.replacingOccurrences(of: "idol-", with: ""))"
    } else if isLEDTemplate {
        return "led_\(id.replacingOccurrences(of: "led-", with: ""))"
    }
    return nil
}
```

## 验证图片是否添加成功

### 方法1：在Xcode中检查
1. 打开 `Assets.xcassets`
2. 查看是否有以下 Image Set：
   - neon_1, neon_2, neon_3, neon_4
   - idol_1, idol_2, idol_3, idol_4
   - led_1, led_2, led_3, led_4

1. 在Xcode中运行应用（Cmd+R）
2. 进入"模版"页面
3. 查看各个分类的卡片

5. 如果显示深色占位符，说明图片未添加或命名错误

## 常见问题

### Q1: 图片不显示怎么办？
**A**: 检查以下几点：
1. 图片是否已添加到 Assets.xcassets
2. Image Set 名称是否正确（不要加 .png）
3. 图片文件是否损坏
4. 重新 Clean Build（Cmd+Shift+K）后再运行

### Q2: 图片显示模糊怎么办？
**A**: 
1. 确保图片尺寸足够大（至少 640×360）
2. 添加 2x 和 3x 版本的高清图
3. 使用 PNG 格式保持清晰度

### Q3: 图片文件太大怎么办？
**A**:
1. 使用在线工具压缩：TinyPNG (https://tinypng.com)
2. 使用Mac应用：ImageOptim
3. 在Photoshop中"存储为Web所用格式"
4. 降低图片质量（JPG格式）

### Q4: 可以使用中文命名吗？
**A**: 
不建议！虽然技术上可以，但：
- 可能导致编码问题
- 不符合iOS开发规范
- 建议使用英文或拼音

### Q5: 需要添加 @2x 和 @3x 版本吗？
**A**:
- 不是必须的，但推荐
- 只添加 1x 版本也可以正常显示
- 添加多个版本可以在不同设备上显示更清晰
