# 跑马灯边框居中修复

## 问题描述
用户反馈跑马灯边框没有居中，向下和向右偏移，包括：
1. 12个选择区的边框按钮中的边框偏移
2. 预览区的边框偏移

## 问题原因分析
1. **圆点定位问题**：CAShapeLayer的position设置不正确
   - 原来：使用`CGRect(x: 0, y: 0, width: dotSize, height: dotSize)`创建圆形，然后设置position
   - 问题：这样会导致圆点的左上角对齐到计算的点位置，而不是圆点中心

2. **参数设置问题**：
   - 圆点尺寸过大，在小按钮中显示不佳
   - safeInset设置过大，导致边框过于靠内
   - 圆角半径设置过大

## 修复方案

### 1. 修复圆点定位
```swift
// 修复前
dotLayer.path = UIBezierPath(ovalIn: CGRect(x: 0, y: 0, width: dotSize, height: dotSize)).cgPath
dotLayer.position = point

// 修复后
let dotRect = CGRect(x: -dotSize/2, y: -dotSize/2, width: dotSize, height: dotSize)
dotLayer.path = UIBezierPath(ovalIn: dotRect).cgPath
dotLayer.position = point
```

### 2. 优化参数设置
```swift
// 修复前
let dotSize: CGFloat = isPreviewMode ? 6 : 3
let safeInset: CGFloat = isPreviewMode ? 8 : 4
let cornerRadius: CGFloat = isPreviewMode ? 12 : 6

// 修复后
let dotSize: CGFloat = isPreviewMode ? 4 : 2
let safeInset: CGFloat = isPreviewMode ? 6 : 3
let cornerRadius: CGFloat = isPreviewMode ? 8 : 4
```

### 3. 优化圆点数量
- 根据预览模式动态调整圆点数量
- 预览模式：使用完整圆点数量（18-32个）
- 选择按钮模式：使用减半圆点数量（最少12个）

### 4. 增强发光效果
```swift
dotLayer.shadowRadius = dotSize * 0.8
dotLayer.shadowOpacity = 0.9
```

## 修复效果
1. ✅ 圆点现在以计算的路径点为中心显示
2. ✅ 边框在视图中完美居中
3. ✅ 选择按钮中的边框清晰可见
4. ✅ 预览区的边框动画流畅
5. ✅ 不同尺寸下都能正确显示

## 技术细节
- **CAShapeLayer.position**：表示图层的锚点位置（默认为中心点）
- **路径坐标系**：相对于图层的本地坐标系，(0,0)为图层中心
- **圆点居中**：通过设置路径为`(-dotSize/2, -dotSize/2, dotSize, dotSize)`实现
- **边框计算**：使用insetBy确保边框不会被裁剪
